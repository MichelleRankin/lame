When the page loads, you should create an instance of the KeyCodeDoorLock to manage the state of each .lock. The constructor requires that you provide an answer keycode as a string. For example, you could provide the string "1234" to create a lock that expects the user to enter 1, then 2, then 3, then 4. The answer keycode for each lock is specified in a data-answer attribute on the lock element.

When the user presses any of the keys on the number pad, enterKey should be called with the appropriate number and the status of the lock should be updated on the page.

When the reset button is pressed, the reset should be called and the status UI should be updated.

When the admin reset button is pressed, the reset button should be called, but you should provide an argument specifying that this is an admin reset. To do so, you provide an object with an admin property with the value true ({admin: true}).

With all of this in place, you should have a working key code lock UI! Your finished UI should do the following things:

Starts off locked

Answer keycode is configured from data-answer

Gets unlocked when the user provides the right code

Requires the user to reset their entered keys if they make a mistake

Locks when reset

Hard locks after three consecutive resets without unlocking

Does not process entered keys while hard locked

Does not process resets while hard locked

Can be reset from a hard lock by an admin

Admin resets reset the consecutive reset count

Admin resets reset the entered keys

Each individual lock handled correctly.
